How to config xdebug

1) config xdebug for debuggear by Browser

Install browser toolbar

Enable listening for PHP Debug Conections

2) webserver configuration file (server_name)

3) Config Web Server in PHP Storm

File -> Settings->Languages & Frameworks -> PHP -> Servers:

Configure absolute path on the Servers

Your computer (host) absolute path on the Servers (inside the container)
/path/to/current/project/ /var/www/html/
/path/to/current/project/pub/ /var/www/html/pub/

4) Set PHP Web Aplication

Run -> Edit Configurations

5) xdebug by console

to use XDEBUG by console set PHP_IDE_CONFIG with same ServerName

Example:

PHP_IDE_CONFIG=serverName=magento2.com

6) file ext-xdebug.ini

Plase run by console: warp php info

open file: .warp/docker/config/php/ext-xdebug.ini

Finally, make sure to configure your xdebug correctly. However you get your xdebug.remote_host into the container, ensure it has similar settings:

For Linux: xdebug.remote_host=172.17.0.1

For Mac OS X: xdebug.remote_host=10.254.254.254

sudo ifconfig lo0 alias 10.254.254.254
sudo curl -o /Library/LaunchDaemons/com.ralphschindler.docker_10254_alias.plist https://gist.githubusercontent.com/ralphschindler/535dc5916ccbd06f53c1b0ee5a868c93/raw/com.ralphschindler.docker_10254_alias.plist
